1.springboot介绍

不需要以前xml，一样可以完成ioc/aop

封装代码—-封装功能—–封装jar包

Spring Boot并不是一门新的技术，它是一个集成框架，本质上就是Spring，用于简化Spring项目的各种繁琐的配置。它的目的是帮助开发人员很容易的创建出独立运行和产品级别的基于 Spring 框架的应用。大部分 Spring Boot 应用只需要非常少的配置就可以快速运行起来。

Spring Boot的特性：

创建可以独立运行的 Spring 应用。

直接嵌入 Tomcat 或 Jetty 服务器，不需要部署 WAR 文件。

提供推荐的基础 POM 文件来简化 Apache Maven 配置。

尽可能的根据项目依赖来自动配置 Spring 框架。

提供可以直接在生产环境中使用的功能，如性能指标、应用信息和应用健康检查。

没有代码生成，也没有 XML 配置文件。

8.springboot配置日志

Spring Boot在所有内部日志中使用Commons Logging，但是默认配置也提供了对常用日志的支持，如：Java Util Logging，Log4J, Log4J2和Logback。每种Logger都可以通过配置使用控制台或者文件输出日志内容

回想直接给大家讲解ceki的故事，所以这里肯定选择logback

Logback是log4j框架的作者开发的新一代日志框架，它效率更高、能够适应诸多的运行环境，同时天然支持SLF4J。

5.springboot线程池配置

使用线程池可以减少线程的创建和销毁，提高性能！！

从刚才的线程测试可以看出，每次启动线程，springboot都会从新创建一个线程，线程不重用，显然效率太低，这是因为spring boot自带线程池过于简单，所以在开发中，都会自己配置线程池的属性

别忘了在使用线程的地方，指定线程池的名字，默认方法名小写

18.restfull风格及拦截

API设计，controller的接口

接口名字问题

student/addStudent

student/insertStudent

student/deleteStudent?id=1 del

httpMethod:get post delete

student/1 delete:(http请求的方式)

student/1 post:增加学生

student/1 get：查询学生

get：请求资源

post：增加资源

delete:删除资源

put：修改资源（传递的参数为完整学生的实体类）

patch:修改资源（修改部分资源）

restfull风格，就是把网络上的数据（数据库的一条数据，网络上的某张图片）看成一个资源

，对该资源具体操作由请求方式指定，不在url中出现动词

拦截的实现方式：

1.注解（之前项目中通过自定义注解完成具体方法的拦截）

2.在权限url表增加一个请求方式字段(delete/post)

3.student/* student/2320 (可以用 正则表达式) AntPathMatcher

?：匹配一个字符 不能是/ 或者.

*：匹配0个或者多个字符，不能是/a这种文件夹

**：任意

在xxxMapping中可以使用
